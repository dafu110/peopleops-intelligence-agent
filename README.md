# 💼 AI-Native 智能人事大管家 & HRBP Agent

本项目是一款针对企业 HCM（人力资本管理）真实业务场景落地设计的 **AI-Native 智能人事多路由 Agent 系统**。项目从 0 到 1 独立打通了前端动态文件解析、后端图状态机流转、上下文感知路由、企业级 RAG 引用溯源以及行政动作工具调用的全栈闭环。

## 🎯 核心核心技术亮点（对标工业级落地诉求）

*   **⚡ 状态机工作流编排（LangGraph）**：智能化打破了传统线性链（Chain）死板、不可逆的缺点。基于 `StateGraph` 构建具备严密节点流转、条件路由与终点闭环的图结构状态机，极大提升了 Agent 的确定性与边界可控性。
*   **🧠 上下文感知大模型中央路由器（Context-Aware Router）**：彻底根治了多轮对话追问时的意图漂移痛点。利用 `temperature=0.0` 的大模型配合最近轮次的对话历史（`history`）作为中央网关，能精准识别如 *“那如果他会 React 呢？”* 等简短追问背后指代的依然是简历匹配意图，而非静态关键词拦截。
*   **🔍 生产级 RAG 答案引用溯源（Citation）**：针对企业制度容错率极低的痛点，重构了本地向量库（Chroma + HuggingFace Embeddings）检索流，手动剥离 Document 的 `metadata` 元数据，在前端实现精准到页码的依据标注（如：*来源：《员工手册测试版.pdf》第 3 页*），彻底消除大模型幻觉。
*   **🚀 语义驱动的工具调用闭环（Tool Calling）**：大模型根据深层语义自主判断是否触发绑定的 `@tool` 行政动作（如系统模拟触发邮件网关发送面试邀约），打通了“从思考到行动”的完整端到端交付。
*   **💻 商业级全栈数据流与 UX 交互**：利用 `pypdf` 真正打通 Streamlit 侧边栏上传 PDF 的实时解析与状态注入；采用最新的大模型响应分流与打字机效果（Streaming）滚动渲染，极大缓解用户等待焦虑。

## 📦 快速本地部署与启动

### 1. 环境克隆与依赖安装

确保您的本地环境已安装 Python 3.10+。在项目根目录下激活您的虚拟环境，并运行：

```bash
pip install -r requirements.txt

```

### 2. 规章知识库配置

在项目根目录下创建 `data/` 文件夹，并将您的企业资产文档命名为 `员工手册测试版.pdf` 放入其中：

```text
hrbp_agent/
├── data/
│   └── 员工手册测试版.pdf

```

### 3. 环境变量配置

在项目根目录下新建 `.env` 文件，配置您的大模型 API 密钥：

```ini
OPENAI_API_KEY="your_deepseek_api_key"
OPENAI_API_BASE="[https://api.deepseek.com/v1](https://api.deepseek.com/v1)"

```

### 4. 一键本地运行

运行 Streamlit 专属防路径混淆命令启动全栈服务：

```bash
python -m streamlit run app.py

```

启动成功后，在浏览器访问 `http://localhost:8501` 即可进入生产级别的交互看板。

## 🧪 核心测试用例验证链路

* **链路一：RAG 规章问答与溯源验证**
* *输入*：“我下周去北京出差，住宿和餐饮能报销多少钱？”
* *预期*：系统通过本地 Embedding 召回文本片段，流式输出回答，并在文末完美附带 `🔍 官方规章参考依据： - 📄 《员工手册测试版.pdf》第 X 页`。


* **链路二：多轮记忆跨上下文追问验证**
* *操作*：左侧上传真实 PDF 简历并粘贴 JD。主对话框输入：“帮我评估一下这份简历”。
* *预期*：系统给出严格符合 JSON 规范的 Pros/Cons Markdown 评估报告。
* *连续追加输入*：“那如果他自学过半年的 React 和前端开发呢？”
* *预期（亮点）*：中央路由器结合 Context 判定其属于能力评估延续，**分值实时动态刷新提升**，且【潜在短板】自动修正，完美解决意图漂移。


* **链路三：Tool Calling 闭环动作验证**
* *输入*：“我觉得这个候选人很合适，帮我给他发送一封明天下午两点的面试邀约邮件。”
* *预期*：Agent 自主决定调用 `send_interview_email` 工具，提取参数，前端打印出 `🚀 [系统动作] 已成功触发邮件网关...` 的成功回执。
